Getting Started:

To get started with the settings, please do the command /settings template to then get a file which you will be editing with whatever you desire! If at any time you're stuck you can use /settings example to get a prefilled template to base your settings on.

Variables:

Name Value Usage
true|false Boolean true = on | false = off
Number Integer Any number from 1 to ∞
String Text Any text you deem suitable for the purpose.
ChannelID | RoleID ID IDs of a channel or a role (remember to keep the " ")

Prefix:

If you have some other bots that have the / as a prefix or you want a different prefix regardless, you have the option to change it. Within the quotation mark (" ") you can put any string or letter to set as your prefix. Also, incase you forgot your prefix, Cryptonix X has a handy feature where you can ping the bot and say prefix (ex. @Cryptonix X#0856 prefix) for the bot to send you it's current prefix.

‎

 "prefix": "/"

Logs:

If you want to look at logs easier or just tired of having to go into server settings to see logs, you can enable the bot's Logging features. Currently available log options are Mod (mute, warn, kick, ban) logs, Join, Leave logs and Message edits/delete logs.

‎

 "logs": true|false,
 "logchannel": "ChannelID",

 "userlog": true|false,
 "uchannel": "ChannelID",

 "messagelog": true|false,
 "mchannel": "ChannelID",

‎

"logs" - Mod Logs: true = enabled | false = disabled

"logchannel" - Mod Log channel: "ChannelID" - replace ChannelID with a suitable channel ID

‎

"userlog" - Join/Leave Logs: true = enabled | false = disabled

"uchannel" - User Log channel: "ChannelID" - replace ChannelID with a suitable channel ID

‎

"messagelog" - Message Logs: true = enabled | false = disabled

"mchannel" - Message Log channel: "ChannelID" - replace ChannelID with a suitable channel ID

Anti Spam:

Whenever people spam your server, Cryptonix X will delete the attempted spam and warn/ban the person who does it multiple times. You can manage it by setting who the bot will ignore with specific roles, the amount of messages before the function kicks in and the announcement message.

‎

 "antispamstatus": true|false,
 "warnBuffer": Number,
 "maxBuffer": Number,
 "maxtime": Number,
 "warnMessage": String,
 "banMessage": String,
 "maxdupwarn": Number,
 "maxdupban": Number,
 "deleteno": Number,
 "exceptionroles": "RoleID1, RoleID2, RoleID3,...",

‎

"antispamstatus" - Anti Spam: true = enabled | false = disabled

"warnBuffer" - Maximum amount of messages allowed to send in the interval time before getting warned.

"maxBuffer" - Maximum amount of messages allowed to send in the interval time before getting banned.

"maxtime" - Amount of time in ms users can send the maxim amount of messages(maxBuffer) before getting banned.

"warnMessage" - Message users receive when warned.

"banMessage" - Message sent in chat when user is banned.

"maxdupwarn" - Maximum amount of duplicate messages a user can send in a timespan before getting warned.

"maxdupban" - Maximum amount of duplicate messages a user can send in a timespan before getting banned.

"deleteno" - Deletes the message history of the banned user in x days.

"exceptionroles" - ID of roles that are exempt from spam filter.

Punish:

Cryptonix X will warn/kick/ban automatically (if enabled) users that got too many warns because lets be real, no one wants them in the server anyways.

‎

 "autopunishstatus": true|false,
 "mutestatus": true|false,
 "mute": Number,
 "kickstatus": true|false,
 "kick": Number,
 "banstatus": true|false,
 "ban": Number,

‎

"autopunishstatus" - true = enabled | false = disabled

"mutestatus" - Mute punish: true = enabled | false = disabled

"mute" - Number of warns before Mute

"kickstatus" - Kick punish: true = enabled | false = disabled

"kick" - Number of warns before Kick

"banstatus" - Ban punish: true = enabled | false = disabled

"ban" - Number of warns before Ban

Self Roles:

Users of Cryptonix X mod bot has the ability to set roles that the members can self assign

‎

 "selfrolestatus": true|false,
 "selfroleID": "roleID1,roleID2,roleID3,...",

‎

"selfrolestatus" - true = enabled | false = disabled

"selfroleID" - ID of roles can be self assigned by members.

Anti Ads:

Cryptonix X can also detect annoying disgusting Ads

‎

 "antiads": true|false,
 "antiadsmessage": "String",
 "antiadsaction": "mute|warn|kick|ban",

‎

"antiads" - true = enabled | false = disabled

"antiadsmessage" - Message sent in the same channel when a user sent an discord.gg link. In the message you can use {USER} to mention the user and {SERVER} to have the guild name in the message.

"antiadsaction" - Action bot will take when a user sent an Ad

Strict Mode:

Strict mode is a feature that can detect recently made account and whether an account has one of those annoying disgusting Ads in them.

‎

 "strictmode": true|false,
 "days": Number,
 "action": "mute|kick|ban",

‎

"strictmode" - true = enabled | false = disabled

"days" - Days in number the account must be created before to bypass Strict Mode

"action" - Action bot will take when a user get detected by Strict Mode

Auto Role:

Give someone a nice role using Auto Role when they join your server if you really feel like you want to colour them to something else than the default, white colour

‎

 "autorolestat": true|false,
 "autorole": "RoleID",

‎

"autorolestat" - true = enabled | false = disabled

"autorole" - ID of a role you want the user to have when they join your guild.

XP System:

Let users level up the more and more they talk in the guild.

‎

 "xpgain": true|false,

‎

"xpgain" - true = enabled | false = disabled

Chat Bot:

Let users chat with Cryptonix X. (The API is currently broken so it doesn't matter whether you enable it or not...)

‎

 "chatbot": true|false,

‎

"chatbot" - true = enabled | false = disabled

Channel Ignore:

Channel Ignore is a feature that makes it so that the bot doesn't respond to any message sent by a normal user (someone who doesn't have MANAGE_ROLES_OR_PERMISSIONS perms.) in a specific channel.

‎

 "channelignorestats": true|false,
 "channelignore": "ChannelID_1,ChannelID_2,ChannelID_3,...",

‎

"channelignorestats" - true = enabled | false = disabled

"channelignore" - IDs of channels for the bot to ignore.

Welcomer:

Give your new members a warm welcoming message by setting up Welcomer

‎

 "welcomer": true|false,
 "welcomerchannel": Number,
 "welcomertext": "String",

‎

"welcomer" - true = enabled | false = disabled

"welcomerchannel" - ID of the channel where the welcomer message will be sent

"welcomertext" - Your custom welcoming message. In the message you can use {USER} to mention the user and {SERVER} to have the guild name in the message.
